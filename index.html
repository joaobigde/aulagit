O que é o Git

Git é um sistema de controle de versão distribuído que permite que desenvolvedores acompanhem mudanças no código durante o desenvolvimento de software. Ele foi criado por Linus Torvalds em 2005 para gerenciar o desenvolvimento do kernel Linux.

O que é o GitHub

GitHub é uma plataforma de hospedagem de código baseada em Git. Ele adiciona recursos próprios como controle de acesso, rastreamento de problemas, solicitações de recursos, gerenciamento de tarefas e wikis para cada projeto.

Diferenças entre Git e GitHub

  • Git é uma ferramenta de linha de comando, enquanto GitHub é uma plataforma web
  • Git é instalado localmente, GitHub é hospedado na nuvem
  • Git gerencia versões do código, GitHub adiciona colaboração
  • Git é open-source e gratuito, GitHub tem planos gratuitos e pagos

Comandos Linux no Git Bash

  • ls Listar arquivos
  • cd Mudar diretório
  • pwd Mostrar diretório atual
  • mkdir Criar pasta
  • rm Remover arquivos
  • cp Copiar arquivos
  • mv Mover/renomear
  • cat Mostrar conteúdo

Comandos Git Essenciais

  • git init Iniciar repositório
  • git clone Clonar repositório
  • git add Adicionar alterações
  • git commit Registrar mudanças
  • git push Enviar para remoto
  • git pull Buscar do remoto
  • git status Verificar estado
  • git log Ver histórico

Criar Repositório no GitHub

Passo 1: Login

Acesse github.com e faça login na sua conta.

Passo 2: Novo repositório

Clique no ícone "+" no canto superior direito e selecione "New repository".

Passo 3: Preencher dados

Digite um nome, descrição e escolha a visibilidade (público ou privado).

Passo 4: Criar

Clique em "Create repository" para finalizar.

Conectar Projeto Existente ao GitHub

Passo 1: Inicializar Git

Execute: git init na pasta do projeto

Passo 2: Adicionar arquivos

Execute: git add . para adicionar tudo

Passo 3: Primeiro commit

Execute: git commit -m "Primeiro commit"

Passo 4: Conectar ao GitHub

Execute: git remote add origin URL_DO_REPOSITORIO

Passo 5: Enviar

Execute: git push -u origin main

Como Gerar uma Versão (Commit)

Passo 1: Verificar status

Execute git status para ver as alterações

Passo 2: Adicionar mudanças

Execute git add . ou git add arquivo.especifico

Passo 3: Fazer commit

Execute git commit -m "Mensagem descritiva"

Enviar para o GitHub

Passo 1: Commit das alterações

Certifique-se de ter feito commit de todas as alterações

Passo 2: Enviar para remoto

Execute git push origin main (ou seu branch)

Passo 3: Verificar

Acesse seu repositório no GitHub para confirmar o envio
